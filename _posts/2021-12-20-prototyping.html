Breadboarding

First, I put all the components on a breadboard.



For adjusting pitch, volume, and speed, I used three 10k potentiometers.

While I initially had a dial to control which samples where being played, I changed it to having several buttons that each correspond to a category to make it easier to switch between categories.

I also have two push buttons to move through samples, and one push button to toggle whether the sound loops.

The Code

There are two components to the code.

The Arduino code gets the values from the components and sends it through the serial ports.

The Max MSP code gets the values through the serial ports and manipulates the parameters of the sample being played with [~sfplay].

All the commented code, including the sound samples, can be found on my Github.

Choosing noises

I decided to use prerecorded samples for the noises, rather than synthesizing my own noises. I thought the noises would sound more "natural" this way.

I used samples from FreeSound.org for the first category of noises, I plan to do the same for the rest of the categories

The Box

I designed the box using Makercase and added icons to the file using Adobe illustrator.

I chose the icons to make them as universal as possible (without being language specific), but I'll definitely need to do some user testing to see if these icons make sense. I also like the idea of this instrument being exploratory, so not all the control's functions need to be immediately apparent.

Then I lasercut the design on cardboard using the Epilog Mini lasercutters.

Next steps...

Some of the code is incomplete, I still have to incorporate the volume, pitch, and speed dials into the MaxMSP code.

I also don't have all the samples chosen for all the noise categories.

Originally, I made the decision to have the boxes all be capable of playing every sample. I'm still debating this - it might be nice to have one "instrument" per noise category, true to the original intonarumori. On the other hand, having these instruments be dynamic could add versatility to the possible noise combinations that could be produced. I'll keep thinking about this.

I started soldering the components together, moving them off the breadboard, but then NYU told us to avoid going to campus, so I'll leave the soldering for when COVID isn't spiking.
